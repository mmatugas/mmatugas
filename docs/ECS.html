Announcement:

Short Delivery Web Applicaton


Hello, readers! I am happy to announce that my next project is to make a web application.
Yes, as you noticed in the title area, I am planning to create a short delivery web application.
It would be called "ECS Delivery." ECS delivery is short for Efficient, Convenient, Secured Delivery.
ECS web app offers the fastest and most convenient way to send packages to everyone as long as
it is close range. Everyone could book a delivery. ECS app is most convenient for online sellers who don’t
have time to make a trip to any delivery services or don’t have time to make errands to deliver their
parcels within the city. Also, ECS web app is not only limited for online sellers,everyone could use it
as long as there’s an item that needs to be delivered. The service will be available on ecsdelivery.com.
The primary functions of the web app are: Users can create an account to book a delivery
or it could be done through guest function. Take note the main goal of the ECS app is to deliver
parcels within close range, the web app will not compete with Fedex or UPS.

Front End

  • The front end has three main colors, Yellow, White, Black.
  • The navigation bar may change depending on the page where the user is in.
    The web app would have a standard horizontal menu bar.
  • Header main menu would have an image and the web application’s name.
  • Web app content would be different depending what page the user is in
    but for the content of the main menu will show the advantages
    of the web application and also the delivery button.
  • The programming languages that will be used are JavaScript, CSS, HTML.
  • The front end frameworks that will be used is bootstrap or more.

Back End

  • One of the back end services is to be able to verify if the given date
    or scheduled time is available.
  • Second is to create accounts for users and will be able to verify if
    the given email is already in.
  • The back end frameworks that will be used are Node JS and Express JS.

Database Requirements

  • Database Requirement would need information about the user and about
    the delivery(sender, receiver, dare, time.. etc).

Everything that will be used is not decided yet. I would only be working independently
and everything is subject to change.
